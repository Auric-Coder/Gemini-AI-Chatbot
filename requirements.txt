===============================================================
  GEMINI AI CHATBOT — Project Requirements
===============================================================


1. PROJECT OVERVIEW
-------------------
Build a full-stack AI chatbot powered by the Google Gemini API.
The project is structured as a learning-focused, portfolio-ready
application with a React frontend and Node.js/Express backend.


2. TECH STACK
-------------------
  Frontend:
    - React 19 (via Vite 6)
    - Tailwind CSS v4
    - Axios (HTTP client)
    - react-markdown + remark-gfm (Markdown rendering)
    - react-syntax-highlighter (Code block formatting)
    - lucide-react (Icons)
    - react-hot-toast (Toast notifications)

  Backend:
    - Node.js (v18+)
    - Express 5
    - @google/genai (Official Google Gemini SDK)
    - cors
    - dotenv

  Dev Tools:
    - npm (package manager)
    - Git (version control)
    - Vite dev server with API proxy


3. FUNCTIONAL REQUIREMENTS
-------------------

  3.1 Chat Interface
    - Users can type a message and press Enter or click Send
    - User messages displayed on the right side
    - AI responses displayed on the left side
    - Typing/loading indicator while waiting for AI response
    - Auto-scroll to the latest message
    - Input cleared and re-focused after sending
    - Shift+Enter for new line in input
    - Send button disabled while request is in progress

  3.2 AI Integration
    - Backend receives user message via POST /api/chat
    - Backend calls Gemini 2.0 Flash model via @google/genai SDK
    - Backend returns AI response as JSON { reply: "..." }
    - API key stored in backend .env file only (never exposed)

  3.3 Message Features
    - Timestamps displayed on every message (HH:MM format)
    - Markdown rendering for AI responses (headings, lists, tables, links)
    - Syntax-highlighted code blocks with language detection
    - Copy button on code blocks and AI responses
    - Multiline response support

  3.4 Chat Management
    - "New Chat" button to clear the conversation
    - Confirmation dialog before clearing
    - Chat history persisted in localStorage
    - Messages restored on page refresh

  3.5 Theme
    - Dark mode (default) and light mode
    - Theme toggle button in navbar
    - Theme preference saved in localStorage
    - Smooth transition between themes

  3.6 Error Handling
    - Graceful handling of API failures
    - User-friendly error messages via toast notifications
    - Handles rate limits (429), auth errors (403), server errors (500)
    - Network connectivity error handling


4. NON-FUNCTIONAL REQUIREMENTS
-------------------

  4.1 Code Quality
    - Clean, modular, well-commented code
    - Reusable React components
    - Separation of concerns (API logic, UI, state)
    - Consistent naming conventions
    - No duplicate code

  4.2 UI/UX
    - Modern, premium design inspired by ChatGPT/Gemini
    - Glassmorphism styling with backdrop blur
    - Smooth animations and transitions
    - Responsive layout (desktop, tablet, mobile)
    - Custom scrollbar styling
    - Inter font from Google Fonts
    - Gradient accent colors (indigo/violet)

  4.3 Performance
    - Vite for fast dev/build
    - Efficient re-renders with React hooks
    - 30-second API timeout
    - Production build under 1.5MB

  4.4 Security
    - API key never sent to frontend
    - CORS middleware on backend
    - Input validation on API endpoint
    - No hardcoded secrets in source code

  4.5 Accessibility
    - Semantic HTML elements
    - ARIA labels on interactive elements
    - Keyboard navigation support (Enter/Shift+Enter)
    - Focus management after actions

        ├── MessageInput.jsx  — Textarea, send button, Enter/Shift+Enter
        ├── CodeBlock.jsx     — Syntax highlighting, copy, language badge
        ├── LoadingIndicator.jsx — Bouncing dots typing animation

        └── ConfirmDialog.jsx — Modal for clear chat confirmation


6. API SPECIFICATION
-------------------

  Endpoint: POST /api/chat

  Request:
    Content-Type: application/json
    Body: {
      "message": "User's message text"
    }

  Success Response (200):
    {
      "reply": "Gemini's generated response text"
    }

  Error Responses:
    400 — Invalid or empty message
    403 — API key permission error
    429 — Rate limit exceeded
    500 — Server/Gemini API error


7. DEPLOYMENT REQUIREMENTS
-------------------

  Frontend (Vercel):
    - Root directory: frontend
    - Build command: npm run build
    - Output directory: dist
    - Update API base URL for production

  Backend (Render):
    - Root directory: backend
    - Build command: npm install
    - Start command: npm start
    - Environment variable: GEMINI_API_KEY
    - Update CORS origin for production

  Environment Variables:
    - GEMINI_API_KEY  (required, backend only)
    - PORT            (optional, default 5000)
